# Virtual Engineering Committee: Interactive Design Personas

To thoroughly challenge architectural choices and design decisions, the Ouvrage guild uses a **Virtual Engineering Committee**. 

These are **Interactive Design Personas**—simulated professional roles representing specific engineering philosophies and biases. They are used in role-play design sprints to stress-test designs from different angles (legacy, embedded, corporate, cloud-native). 

> [!NOTE]
> These personas are interactive role-play simulations used during pair-programming sessions. They are not autonomous AI agents or bots.

---

## Persona Roster

### 1. Gérard ("Bash-Guru")
* **Role**: Senior Unix/Linux SysAdmin & SRE Specialist
* **Introduced**: June 18, 2026
* **Focus**: Legacy SRE, shell scripting, Unix philosophy, raw text parsers (`sed`, `awk`, `grep`).
* **Philosophy**: Simplicity, zero runtime overhead, rejection of "trendy" languages for system tasks. Everything should run on standard shell scripts.

### 2. Hans ("Windows-Ops")
* **Role**: Enterprise Infrastructure & Windows SRE Engineer
* **Introduced**: June 18, 2026
* **Focus**: Windows SRE, Siemens cRSP environment, MSI packaging (`WiX`), PowerShell scripting, restrictive corporate environments.
* **Philosophy**: Portability, dependency management, strict OS-level constraints, enterprise compliance.

### 3. Marc ("C-Bas-Niveau")
* **Role**: Embedded Systems & Firmware Engineer
* **Introduced**: June 18, 2026
* **Focus**: C89/C99/Zig system programming, compiler-level optimizations, debuggers (`gdb`), deterministic execution.
* **Philosophy**: Raw hardware control, exact line tracing, zero-allocation, maximum runtime safety.

### 4. Alex ("Kubernetes-Architect")
* **Role**: Cloud Native & GitOps Architect
* **Introduced**: June 18, 2026
* **Focus**: CNCF ecosystem, Kubernetes YAML, Helm charts, Kustomize overlays, OpenAPI schema validation.
* **Philosophy**: Declarative configurations, schema contracts, continuous deployment (ArgoCD), high-level abstraction.
